v2.0.11 — Important data-integrity hotfix, plus polish.

If you've run more than one Fix to the same Library Drive on a previous version of PDR, a startup cleanup routine has been silently shrinking your indexed library — treating each new Fix run as a "duplicate" of the previous one and deleting database rows for the older runs. Your photos on disk were never affected, but the database forgot about thousands of them, which caused Search & Discovery, Memories and Library Drive Manager to all under-report your real library.

v2.0.11 disables that cleanup. The catch-up indexer added in v2.0.9 (Dashboard banner — "your libraries aren't fully searchable yet — Index now") will rebuild the missing database rows from your photos on disk in one click. AI faces, tags and Takeout-imported album memberships for the cascade-deleted rows would need to be re-derived (re-run AI, re-import Takeout albums) to fully restore.

Library Drive Manager is now structurally the source of truth: the Workspace no longer holds a separate copy of your Library Drive path. The Dashboard, the offline banner, source-add, Fix runs and the title-bar Library pill all read live from the LDM, so removing or clearing sources can no longer make the Dashboard prompt you to pick a drive that you've already attached.

Two related Library Drive behaviors also land in this release:

• Picking a Library Drive in the Library Drive Manager now atomically updates both the live attach state AND the persisted setting that source-add uses, so the two can never drift apart and cause source-add to ignore the drive you've selected. A one-time startup sync also pulls them back into alignment for anyone whose state had already drifted on a previous version.

• Adding a source with no Library Drive set up no longer silently routes the extraction to the system drive. PDR now refuses the source-add with a clear message and opens the Library Drive Manager so you can attach a library and try again.

Polish:

• Minimizing the Analyzing modal now drops you to the same rich amber pill style that Fix uses — phase (Unpacking / Processing / Preparing), percent, elapsed time, and an Open button to bring the modal back. Same muscle memory as a Fix in progress.

• A small "Reclaimed X GB of extracted files" toast appears when you pick Clear Sources from the post-fix prompt (or clear them later). Extracted Takeouts now stay alive while a source is in the Source Menu, so picking Keep Sources leaves the source fully functional for another Fix or re-analysis.

• Orphaned source rows are now cleaned up at launch. If a previous build deleted an extraction without removing the source row (or you manually cleaned PDR_Temp between sessions) the dead source row used to sit in your Source Menu pointing at nothing. PDR now detects that on mount and drops those rows with a single toast. Folder and drive sources are exempt; a disconnected source drive doesn't count as orphan because Fix copies from the extraction on your Library Drive, not from the original archive.

• Quick Access in Add Source is now expanded by default. The Desktop / Documents / Pictures / Downloads + saved-locations shortcut row is open the moment the picker appears, so it's not hidden behind a chevron. Still collapsible if you want more room for the All Drives grid.

• Keyboard receivers, empty card-reader slots and CD/DVD drives no longer show up as Library Drives. Windows assigns a drive letter to any device with a tiny embedded chip, so peripherals like wireless-keyboard receivers used to clutter the Pick Library Drive picker (flagged "Too small" but still visible). PDR now hides removable drives smaller than 1 GB and CD/DVD drives outright. Real removable storage like USB sticks and SD cards stays.

• Spelling across the app is now consistent American English — words like "analyze", "organize", "recognize", "color" and "favorite" read the same way everywhere they appear, in modals, tooltips, the Quick Tour, Best Practices, and About PDR.

This update will install automatically the next time you close PDR.
