v2.0.9 — Two fixes.

First: the photo viewer is working again. v2.0.8 shipped with a path bug that stopped the dedicated viewer window from loading in installed builds (it worked in development). Photos and videos opened from Search & Discovery, Memories, and Albums display correctly again. No data was at risk — the viewer just refused to render.

Second: catch-up indexer for older libraries. Auto-indexing has been on by default since v2.0.5, so anyone whose library was Fixed on v2.0.4 or earlier (or who turned indexing off) has photos on disk that don't appear in Search & Discovery, Memories, or the Date Editor. PDR now detects that gap on your Dashboard and offers a one-click backfill using the same non-destructive re-indexer the Parallel Library uses — existing rows are left alone; only new files get added.

This update will install automatically the next time you close PDR.
