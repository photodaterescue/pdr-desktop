v2.0.9 — Viewer rebuilt, Search & Discovery sped up, Memories polished.

The PDR Viewer. The bottom-of-screen filmstrip used to load every photo in the bucket at full resolution the moment you opened the viewer — which on a big Memories month with thousands of photos pegged the graphics card and could blank the screen. Each tile now pulls a small cached thumbnail, only the tiles near the visible area load at all, and the main image is hidden until it's correctly fitted to the window (no more "zoomed-in corner" flash on slow loads). Mouse-wheel scroll over the strip now scrolls it horizontally, the same way People Manager works. Rapid double-clicks on a tile no longer log "failed to load" errors. And the viewer-path bug from v2.0.8 is genuinely fixed this time — the original patch missed one code branch.

Search & Discovery. Replaces the "Load more" button at the bottom of the results with proper infinite scroll — the next page lands automatically as you scroll, with a small "Showing N of M" line at the bottom and a final "Showing all N results" when you reach the end. The page itself opens faster: the dropdown options for every filter now load just after S&D paints, not before, and the live filter counts no longer recompute on every keystroke when you've got no active filters. And the results grid is virtualised — after multiple infinite-scroll pages, only the visible tiles stay in the browser's memory instead of accumulating.

Memories — By Date. The summary line, library selector, density toggle, and "Jump to latest" button all moved up onto the same row as the By Date / Albums toggle, freeing a row's worth of vertical space for the actual photos. The month header now reads "X photos · Y videos" instead of "X files". The "On This Day" AI suggestion card can be hidden with an X in its corner, restored from a small lavender pill in the controls row.

Background fix. A pre-existing duplicate-row consolidation pass at startup didn't know about the Albums feature (added in v2.0.8) and could silently drop album memberships when it merged duplicate file records. Fixed — the consolidator now copies album memberships and re-points album covers onto the kept row before deleting the duplicate, so this can never happen again on your library going forward.

This update will install automatically the next time you close PDR.
