v2.0.12 — Emergency follow-up to v2.0.11. Fixes the packaging bug that broke v2.0.11's background workers, and adds a one-click recovery path for anyone whose library was damaged by an older version's startup cleanup.

What happened: v2.0.11 packaged its five background worker scripts (ai-worker, cleanup-worker, conversion-worker, extract-worker, startup-worker) inside the app's compressed bundle, where the operating system can't see them. The result: workers crashed on launch with "module not found", which broke the post-Fix cleanup, AI face detection, ZIP extraction, format conversion, and startup database housekeeping. v2.0.11 was pulled from the public download immediately on discovery; this v2.0.12 installer is the one to use.

Recovery for v2.0.10 upgrade victims:

If you upgraded from v2.0.10 in the last 10 days and noticed photos disappearing from Search & Discovery / Memories / Albums, this is for you. v2.0.10's startup cleanup had a bug that treated multiple Fix runs to the same Library Drive as duplicates and cascade-deleted the older runs from the database. Photos on disk were never touched — only the database rows.

v2.0.12 detects this damage at launch. If your Library Drive holds a database backup with significantly more photos than the local index, a yellow banner appears on the Dashboard offering a one-click restore. Click "Restore now" and PDR pulls the backup back into the local database, including all album memberships, AI face matches, and tags. The page reloads and everything you'd lost is back. No re-import, no re-AI, no re-Fix.

Defence in depth:

Going forward, every PDR launch now snapshots the local database to your Library Drive's hidden ".pdr" folder BEFORE running any startup cleanup. If a future cleanup ever misbehaves, the pre-cleanup snapshot is sitting right there for the recovery banner to offer. This is on top of v2.0.11's fix to the original bug — the cascade-cleanup itself is disabled — so this is a belt-and-braces safety net rather than the primary defence.

Other fixes:

• The release pipeline now kills any running PDR / Electron processes before packaging, so a developer's dev session can never lock files that the build needs to overwrite. Closes the door on a class of subtle packaging defects.

This update will install automatically the next time you close PDR. If you're currently on v2.0.11 and the auto-update doesn't fire, download the v2.0.12 installer directly from updates.photodaterescue.com.
