v2.0.15 — A Recycle Bin, a working Photo Format converter, and a smoother end-of-Fix experience.

This is the closing release for v2.0.x. A quality-of-life pass: a proper Recycle Bin, the Photo Format converter unlocked and made usable, source-path visibility in the sidebar, and the removal of the freezes and chime stalls that used to bookend every Fix.

Recycle Bin:

Deleted files now go to a recoverable PDR Recycle Bin instead of straight to disk. You can restore them, permanently delete a selection, or empty the bin entirely. Empty-Bin offers a "Skip Windows Recycle Bin" option so you can actually reclaim disk space — by default Windows keeps a full byte-for-byte copy of every deleted file in its own Recycle Bin until that's emptied too, so a permanent delete from PDR alone doesn't free any space. The new checkbox routes files through fs.unlink instead, making the delete genuinely permanent.

Photo Format conversion unlocked and usable:

The Photo Format dropdown on the dashboard was previously gated as "Coming in v2.1". It's now live with PNG and JPG output options. The PNG path runs in a persistent encode worker that's spawned once per Fix and reused across batches (instead of paying a 6-second fork+sharp-load cost on every batch), embeds EXIF dates directly into the encode pipeline (replacing a 35-minute serial exiftool pass on typical large Fixes), and uses faster libvips compression. An infinite-loop hang on near-duplicate files that share the same EXIF date is fixed.

Source rows show where they came from:

Hover any source in the sidebar to see its full filesystem path in a tooltip; right-click for "Show in File Explorer" and "Copy filepath". This disambiguates sources that share the same leaf folder name (two "Camera Roll" folders on different drives, an OneDrive sync folder vs an OneDrive Takeout extract, etc.).

End-of-Fix smoothness:

The several-second hang at the end of every Fix is gone. Catalogue and CSV generation now happens in a dedicated worker thread with per-report chunk caching, off the main process. The completion chime has been moved to the WebAudio API and pre-warmed at app startup, killing the four-second play() stalls that previously made the end of a Fix feel laggy.

Drives panel always populated:

The "Add Source → Drive" surface used to occasionally appear empty when PowerShell's drive query timed out. Added a Node-native fallback (fs.statSync across A-Z) that runs immediately if the PowerShell call doesn't respond within a few hundred milliseconds, so the panel always shows your drives.

Responsiveness:

Indexer, prescan, and analysis now all run off the main thread. Adding a source no longer locks the UI; the white-titlebar "Not Responding" flash is gone. Geocoder (city / country lookup) initialisation is async too, so the workspace opens faster.

Source Menu polish:

Memories and Recycle Bin sidebar icons brightened to match the other sidebar tints. The "Change source" action in the right-click menu now opens the unified folder browser (consistent with the main Add Source flow) instead of the legacy Select Source Type modal.

Settings → After Fix toggles fixed:

"Clear sources after Fix" was previously a no-op that did nothing when you completed a Fix. It now actually clears the source list. The Clear-Sources confirmation prompt before the action also returned this release.

This update will install automatically the next time you close PDR.
