v2.0.10 — Hotfix for the "Library Drive not set" error on source-add.

A small but consequential fix. The Library Drive Manager and the source-add flow were reading the active Library Drive from two different places — the LDM uses the live attach state (most authoritative), the source-add flow was reading from a legacy persisted setting. These two could quietly drift apart after switching libraries, with the result that adding a Takeout would refuse with a "Library Drive — not set" modal even though the LDM clearly showed the drive as active.

Source-add now reads the same source of truth the LDM uses, so what you see in the Library Drive Manager is what every operation acts on. Same family of issue as Jane's sleep-state bug fixed in v2.0.9, but a separate code path that wasn't caught by that work.

This update will install automatically the next time you close PDR.
